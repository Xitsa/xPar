# Шаг 13: Перенос normalbreaks — выбор разрывов строк (без justification)

## Цель шага

Реализовать метод `NormalBreaks` в `ReformatModule` — основной алгоритм выбора разрывов строк
для режима без justification (`just=0`). Этот метод использует `SimpleBreaks` как вспомогательный
и применяет три этапа оптимизации согласно par.doc.

## Контекст

В оригинале `normalbreaks` (reformat.c строки 138–196) — статическая функция,
которая выбирает разрывы строк согласно политике из par.doc для `just=0`:

1. Если `fit=1`: найти минимальную разницу между кратчайшей и длиннейшей строкой,
   перебирая `tryL` от L вниз.
2. Определить максимальную длину кратчайшей строки через `simplebreaks`.
3. Минимизировать сумму квадратов отклонений от `target`.

Этот метод вызывается из `reformat()` (строка 436):
```c
if (just) justbreaks(head,tail,L,last,errmsg);
else normalbreaks(head,tail,L,fit,last,errmsg);
```

## Оригинал (reformat.c строки 138–196)

```c
static void normalbreaks(
  word *head, word *tail, int L, int fit, int last, errmsg_t errmsg
)
{
  word *w1, *w2;
  int tryL, shortest, score, target, linelen, extra, minlen;

  *errmsg = '\0';
  if (!head->next) return;

  target = L;

  /* Этап A: Determine minimum possible difference between */
  /* the lengths of the shortest and longest lines: */
  if (fit) {
    score = L + 1;
    for (tryL = L;  ;  --tryL) {
      shortest = simplebreaks(head,tail,tryL,last);
      if (shortest < 0) break;
      if (tryL - shortest < score) {
        target = tryL;
        score = target - shortest;
      }
    }
  }

  /* Этап B: Determine maximum possible length of the shortest line: */
  shortest = simplebreaks(head,tail,target,last);
  if (shortest < 0) {
    sprintf(errmsg,impossibility,1);
    return;
  }

  /* Этап C: Minimize the sum of the squares of the differences */
  /* between target and the lengths of the lines: */
  w1 = tail;
  do {
    w1->score = -1;
    for (linelen = w1->length,  w2 = w1->next;
         linelen <= target;
         linelen += 1 + isshifted(w2) + w2->length,  w2 = w2->next) {
      extra = target - linelen;
      minlen = shortest;
      if (w2)
        score = w2->score;
      else {
        score = 0;
        if (!last) extra = minlen = 0;
      }
      if (linelen >= minlen  &&  score >= 0) {
        score += extra * extra;
        if (w1->score < 0  ||  score <= w1->score) {
          w1->nextline = w2;
          w1->score = score;
        }
      }
      if (!w2) break;
    }
    w1 = w1->prev;
  } while (w1 != head);

  if (head->next->score < 0)
    sprintf(errmsg,impossibility,2);
}
```

## Алгоритм (пошагово)

### Этап A: Оптимизация ширины (только при fit=1)

Цель: найти такую ширину `target ≤ L`, при которой разница между самой короткой
и самой длинной строкой минимальна.

- `score = L + 1` (начальное значение — хуже любого возможного)
- Перебираем `tryL` от L вниз:
  - Вызываем `simplebreaks(head, tail, tryL, last)` → `shortest`
  - Если `shortest < 0` — есть слово шире `tryL`, дальше нет смысла
  - Если `tryL - shortest < score` — нашли лучшую разницу:
    - `target = tryL`, `score = tryL - shortest`
- Цикл продолжается до бесконечности, пока `simplebreaks` не вернёт -1

**Результат:** оптимальный `target` (может быть < L при `fit=1`, или = L при `fit=0`).

### Этап B: Определение минимальной длины строки

- Вызываем `simplebreaks(head, tail, target, last)` → `shortest`
- Если `shortest < 0` — ошибка (невозможно уложить слова)
- `shortest` — максимальная возможная длина кратчайшей строки при ширине `target`

**Результат:** `shortest` — нижняя граница длины строки.

### Этап C: Минимизация суммы квадратов отклонений

DP с конца к началу (как в `simplebreaks`, но с другой функцией стоимости):

Для каждого слова `w1` (от tail к head):
- `w1->score = -1` (по умолчанию — невозможно)
- Для каждого кандидата `w2` (конец строки):
  - `linelen` — длина строки от w1 до w2 (включительно)
  - `extra = target - linelen` — «лишнее» место в строке
  - `minlen = shortest` (минимальная допустимая длина)
  - Если `w2 == null` (последняя строка):
    - `score = 0`
    - Если `!last`: `extra = minlen = 0` (последняя строка не учитывается)
  - Иначе: `score = w2->score`
  - Если `linelen >= minlen && score >= 0`:
    - `score += extra * extra` (сумма квадратов)
    - Если `score <= w1->score` (лучше или равно):
      - `w1->nextline = w2`
      - `w1->score = score`

**Функция стоимости:** сумма квадратов отклонений от `target`.
Меньше = лучше (строки ближе к `target`).

### Проверка ошибки

Если `head->next->score < 0` — ни один вариант не подошёл, ошибка.

## Адаптация для xpar

### Ключевые отличия

1. **Список слов** — `List<Word>`, а не связный список.
   - `w1->prev` → `words[idx - 1]`
   - `w1->next` → `words[j]`
   - `w1->nextline` → `w1.NextLine` (int?)
   - `head->next` → `words[0]`

2. **Этап A** — вызов `SimpleBreaks` с разными `tryL`.
   - `SimpleBreaks` уже перезаписывает `Score` и `NextLine` — это нормально,
     на этапе C они будут пересчитаны.

3. **Длина строки** — визуальная ширина:
   - `linelen += 1 + isshifted(w2) + w2->length`
   - → `linelen += 1 + (w2.Flags.Shifted ? 1 : 0) + w2.Width`

### План реализации

```csharp
public static string? NormalBreaks(List<Word> words, int L, bool fit, bool last)
{
    if (words.Count == 0) return null;

    int target = L;

    // === Этап A: Оптимизация ширины (только при fit=1) ===
    if (fit)
    {
        int bestScore = L + 1;
        for (int tryL = L; ; tryL--)
        {
            int shortest = SimpleBreaks(words, tryL, last);
            if (shortest < 0) break;
            if (tryL - shortest < bestScore)
            {
                target = tryL;
                bestScore = target - shortest;
            }
        }
    }

    // === Этап B: Определение минимальной длины строки ===
    int shortest = SimpleBreaks(words, target, last);
    if (shortest < 0)
        return "Cannot format paragraph (impossibility 1)";

    // === Этап C: Минимизация суммы квадратов отклонений ===
    int n = words.Count;
    for (int idx = n - 1; idx >= 0; idx--)
    {
        words[idx].Score = -1;

        int linelen = words[idx].Width;
        int j = idx + 1;

        while (j < n && linelen <= target)
        {
            int extra = target - linelen;
            int minlen = shortest;
            int score;

            if (j < n)
            {
                score = words[j].Score;
            }
            else
            {
                score = 0;
                if (!last)
                    extra = minlen = 0;
            }

            if (linelen >= minlen && score >= 0)
            {
                score += extra * extra;
                if (words[idx].Score < 0 || score <= words[idx].Score)
                {
                    words[idx].NextLine = j < n ? j : null;
                    words[idx].Score = score;
                }
            }

            if (j >= n) break;

            linelen += 1 + (words[j].Flags.HasFlag(WordFlags.Shifted) ? 1 : 0) + words[j].Width;
            j++;
        }

        // Обработка случая, когда все оставшиеся слова помещаются на одну строку
        if (j >= n && linelen <= target)
        {
            int extra = target - linelen;
            int minlen = shortest;
            int score = 0;
            if (!last)
                extra = minlen = 0;

            if (linelen >= minlen && score >= 0)
            {
                score += extra * extra;
                if (words[idx].Score < 0 || score <= words[idx].Score)
                {
                    words[idx].NextLine = null;
                    words[idx].Score = score;
                }
            }
        }
    }

    if (words[0].Score < 0)
        return "Cannot format paragraph (impossibility 2)";

    return null;
}
```

**Примечание:** в оригинале цикл `for (linelen = w1->length, w2 = w1->next; ...)` включает
случай `w2 == NULL` (все оставшиеся слова на одной строке). В моей реализации это
обрабатывается отдельно после цикла `while`.

## Результат шага

### Реализованные методы

- `ReformatModule.NormalBreaks(List<Word> words, int L, bool fit, bool last)` — публичный метод.
  - Аналог `normalbreaks()` из reformat.c (строки 138–196).
  - Три этапа:
    1. **Оптимизация ширины (fit):** перебор `tryL` от L вниз, поиск минимальной разницы.

  - Заполняет поля `Score` и `NextLine` для каждого слова.
  - Возвращает `null` при успехе или сообщение об ошибке.
