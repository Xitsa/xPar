# Шаг 16: Интеграция всех частей в метод Reformat

## Цель шага

Завершить реализацию метода `ReformatModule.Reformat`, связав все ранее реализованные
методы в единый конвейер форматирования абзаца.

## Контекст

На предыдущих шагах реализованы все компоненты:

| Шаг | Метод | Назначение |
|---|---|---|
| 11 | `ExtractWords` | Выделение слов, обработка guess, проверка/разбиение длины |
| 12 | `SimpleBreaks` | Максимизация кратчайшей строки (вспомогательный) |
| 13 | `NormalBreaks` | Выбор разрывов строк для just=0 |
| 14 | `JustBreaks` | Выбор разрывов строк для just=1 |
| 15 | `ConstructLines` | Построение выходных строк с префиксами/суффиксами |

Метод `Reformat` (h_009.md) — заглушка, возвращающая `Array.Empty<string>()`.

## Алгоритм (пошагово)

### Шаг 1: Извлечение параметров

```csharp
int prefix = affixes.Prefix;
int suffix = affixes.Suffix;
int width = options.Width;
int L = width - prefix - suffix;
```

### Шаг 2: Выделение слов

```csharp
var extractResult = ExtractWords(
    segments, startIndex, endIndex,
    prefix, suffix, width,
    options.TerminalChars,
    options.Cap, options.Guess, options.Report);

if (extractResult.ErrorMessage != null)
    throw new InvalidOperationException(extractResult.ErrorMessage);

var words = extractResult.Words;
var prefixes = extractResult.Prefixes;
var suffixes = extractResult.Suffixes;
L = extractResult.L;
```

### Шаг 3: Выбор разрывов строк

```csharp
string? breakError;
if (options.Just)
    breakError = JustBreaks(words, L, options.Last);
else
    breakError = NormalBreaks(words, L, options.Fit, options.Last);

if (breakError != null)
    throw new InvalidOperationException(breakError);
```

### Шаг 4: Построение выходных строк

```csharp
bool touch = options.Touch ?? (options.Fit || options.Last);

var lines = ConstructLines(
    extractResult, affixes,
    options.Hang, options.Just, options.Last, touch);

return lines.ToArray();
```

## Соответствие параметров `par` и `xpar`

| Параметр `par` | `ParOptions` | Куда передаётся |
|---|---|---|
| `width` | `options.Width` | `ExtractWords` → L |
| `prefix` | `affixes.Prefix` | `ExtractWords`, `ConstructLines` |
| `suffix` | `affixes.Suffix` | `ExtractWords`, `ConstructLines` |
| `cap` | `options.Cap` | `ExtractWords` (guess) |
| `guess` | `options.Guess` | `ExtractWords` |
| `report` | `options.Report` | `ExtractWords` |
| `fit` | `options.Fit` | `NormalBreaks` |
| `just` | `options.Just` | выбор NormalBreaks/JustBreaks |
| `last` | `options.Last` | NormalBreaks/JustBreaks, ConstructLines |
| `touch` | `options.Touch` | ConstructLines (по умолчанию `fit \|\| last`) |
| `hang` | `options.Hang` | ConstructLines |
| `terminalChars` | `options.TerminalChars` | `ExtractWords` |

## План шага

### 16.1 Реализация `Reformat`

- Заменить заглушку на полную реализацию.
- Последовательные вызовы: ExtractWords → NormalBreaks/JustBreaks → ConstructLines.
- Обработка ошибок через `ErrorMessage` / возвращаемую строку.

### 16.2 Добавление тестов

- Тест: justification (just=true).

- Тест: hang > 0 — дополнительные строки.

- Тест: ошибка — строка короче prefix+suffix.

## Результат шага

### Реализация `Reformat`

- Метод `ReformatModule.Reformat(...)` полностью реализован.
- Конвейер: ExtractWords → NormalBreaks/JustBreaks → ConstructLines.
- Обработка ошибок через `InvalidOperationException`.
- `touch` по умолчанию = `options.Fit || options.Last`.
- `TerminalChars` по умолчанию = `.?!:` (если не задан в options).

### Добавленные тесты (ReformatTests.cs)

- 13 интеграционных тестов:
  - Одна строка, без изменений.
  - Базовое форматирование, разрывы не нужны.
  - Нужны разрывы строк.
  - Justification (just=true).
  - Fit — оптимизация ширины.
  - Last — последняя строка учитывается.
  - Hang — дополнительные строки.
  - Префикс и суффикс копируются.
  - Guess — слияние curious/capital слов.
  - Пустой IP → пустой результат.
  - Одна строка в IP.
  - Ошибка — слово слишком длинное (report=true).
  - Unicode (эмодзи).
