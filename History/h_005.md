# Шаг 5: Реализация Compresuflen — вычисление общего префикса и суффикса

## Цель шага

Реализовать аналог `compresuflen()` из оригинального `par` — метод, который принимает набор строк
и вычисляет `comprelen` (длину общего префикса) и `comsuflen` (длину общего суффикса),
работая с графем-кластерами вместо байт.

## План шага

### 1. Теоретическая основа (из par.doc)

**comprelen:**
> Given a non-empty sequence <S> of lines, let <c> be their longest common prefix.
> If the parameter <body> is 0, place a divider just after the leading non-body characters in <c>
> (at the beginning if there are none). If <body> is 1, place the divider just after the last
> non-space non-body character in <c> (at the beginning if there is none), then advance the divider
> over any immediately following spaces. The comprelen of <S> is the number of characters preceding
> the divider.

**comsuflen:**
> Given a non-empty sequence <S> of lines, let <p> be the comprelen of <S>.
> Let <T> be the set of lines which results from stripping the first <p> characters from each line in <S>.
> Let <c> be the longest common suffix of the lines in <T>. If <body> is 0, place a divider just before
> the trailing non-body characters in <c> (at the end if there are none), then advance the divider over
> all but the last of any immediately following spaces. If <body> is 1, place the divider just before
> the first non-space non-body character, then back up the divider over one immediately preceding space
> if there is one. The comsuflen of <S> is the number of characters following the divider.

### 2. Ключевые различия (оригинал vs xpar)

| Аспект | Оригинал (par.c) | xpar |
|--------|-------------------|------|
| Единица измерения | `char` (1 байт) | Графем-кластер |
| Индексация | `line[pre]` — прямой доступ по указателю | Поиск позиции N-го графема через `StringInfo` |
| Возвращаемое значение | Разница указателей (байты) | Количество графем |
| Сравнение символов | `*p1 == *p2` | Сравнение графем-кластеров (строк) |
| `csmember` | Проверка одного `char` | `Charset.IsMember(grapheme)` |

### 3. Тип результата — record `CompresuflenResult`

```csharp
/// <summary>
/// Результат вычисления comprelen и comsuflen (в графемах).
/// </summary>
public readonly record struct CompresuflenResult(int Prefix, int Suffix);
```

Используется `record struct` — value type, иммутабельный, с value-based равенством.

### 4. Сигнатура метода

```csharp
/// <summary>
/// Вычисляет comprelen и comsuflen для набора строк.
/// </summary>
/// <param name="segments">Массив сегментов строк (LineSegment)</param>
/// <param name="startIndex">Индекс первого сегмента для обработки (включительно)</param>
/// <param name="endIndex">Индекс последнего сегмента для обработки (включительно)</param>
/// <param name="bodyChars">Набор body-символов</param>
/// <param name="body">Флаг body режима (false = 0, true = 1)</param>
/// <param name="minPrefix">Минимальная известная длина префикса (в графемах)</param>
/// <param name="minSuffix">Минимальная известная длина суффикса (в графемах)</param>
/// <returns>CompresuflenResult с полями Prefix (comprelen) и Suffix (comsuflen) в графемах</returns>
public static CompresuflenResult Compresuflen(
    IReadOnlyList<LineSegment> segments,
    int startIndex,
    int endIndex,
    Charset bodyChars,
    bool body,
    int minPrefix,
    int minSuffix)
```

### 5. Алгоритм (пошагово, адаптация par.c:481-540)

#### Часть A: Вычисление префикса (comprelen)

**Шаг A1.** (par.c 492-493)
```
start = lines[0]  // первая строка
knownStart = позиция char после minPrefix графем в start
end = knownStart  // изначально end = knownStart
```

**Шаг A2.** (par.c 494-497) — найти конец «тела» первой строки
```
if body:
    // body=1: идти до конца строки (все символы — тело)
    while (текущий графем != конец строки) ++end
else:
    // body=0: идти до первого body-символа
    while (текущий графем != конец && !bodyChars.IsMember(графем)) ++end
```

**Шаг A3.** (par.c 498-502) — сузить end по всем строкам
```
for lineIdx = startIndex+1 .. endIndex:
    p1 = knownStart (в графемах lines[startIndex])
    p2 = minPrefix в графемах текущей строки
    while p1 < end и графем(p1) == графем(p2):
        ++p1, ++p2
    end = p1  // сужаем — общий префикс не может быть больше
```

**Шаг A4.** (par.c 503-510) — корректировка для body=1
```
if body:
    // Отступить от end до knownStart, ища последний non-space non-body символ
    for p1 = end; p1 > knownStart:
        --p1
        if графем(p1) != " ":
            if bodyChars.IsMember(графем(p1)):
                end = p1  // включаем этот символ
            else:
                break  // non-body — выходим, end уже за ним
```

**Шаг A5.** (par.c 511)
```
result.Prefix = количество графем от start до end
```

#### Часть B: Вычисление суффикса (comsuflen)

**Шаг B1.** (par.c 513-515)
```
knownStart = позиция после result.Prefix графем в lines[startIndex]
end = количество графем в lines[startIndex]
knownend = end - minSuffix
```

**Шаг B2.** (par.c 516-520) — найти start (начало «тела» суффикса)
```
if body:
    start = knownStart  // body=1: всё между knownStart и knownend — тело
else:
    // body=0: идти назад от knownEnd, пропуская non-body символы
    start = knownEnd
    while start > knownStart && !bodyChars.IsMember(графем перед start):
        --start
```

**Шаг B3.** (par.c 521-527) — сузить start по всем строкам
```
for lineIdx = startIndex+1 .. endIndex:
    knownstart2 = result.Prefix в текущей строке
    p2 = конец текущей строки
    p1 = knownEnd, p2 отступить на minSuffix графем от конца
    while p1 > start && p2 > knownstart2 && графем(p1-1) == графем(p2-1):
        --p1, --p2
    start = p1  // сужаем
```

**Шаг B4.** (par.c 528-535) — корректировка для body=0 и body=1
```
if body:
    // Идти вперёд от start, пока space или body-символ
    p1 = start
    while start < knownend && (графем(start) == " " || bodyChars.IsMember(графем(start))):
        ++start
    if start > p1 && графем(start-1) == " ":
        --start  // отступить на один пробел назад
else:
    // Пропустить двойные пробелы в начале суффикса
    while (end - start >= 2) && графем(start) == " " && графем(start+1) == " ":
        ++start
```

**Шаг B5.** (par.c 536)
```
result.Suffix = количество графем от start до end
```

### 5.5. Соответствие переменных оригинала и xpar

| Переменная в par.c | Тип | Назначение | Переменная / выражение в xpar |
|---|---|---|---|
| `start` | `const char*` | Указатель на начало первой строки | `graphemes[0]` (массив графем первой строки), индекс 0 |
| `end` | `const char*` | Указатель на конец общего префикса (временно) / конец строки (для суффикса) | `graphemeIndex` — индекс в массиве графем |
| `knownstart` | `const char*` | Позиция после `pre` байт в первой строке | `minPrefix` — индекс в массиве графем |
| `knownend` | `const char*` | Позиция за `suf` байт до конца первой строки | `graphemes.Length - minSuffix` |
| `knownstart2` | `const char*` | Позиция после `*ppre` байт в текущей строке | `result.Prefix` — индекс в массиве графем текущей строки |
| `p1` | `const char*` | Указатель при сканировании первой строки | `i1` — индекс графема в массиве графем первой строки |
| `p2` | `const char*` | Указатель при сканировании текущей строки | `i2` — индекс графема в массиве графем текущей строки |
| `line` | `const char * const *` | Итератор по массиву строк | `lineIdx` — индекс в `lines`, цикл `for` |
| `*ppre` | `int*` | Выходной параметр — comprelen | `result.Prefix` |
| `*psuf` | `int*` | Выходной параметр — comsuflen | `result.Suffix` |
| `bodychars` | `charset*` | Набор body-символов | `bodyChars` (параметр `Charset`) |
| `body` | `int` | Флаг режима body (0 или 1) | `body` (параметр `bool`, соответствует `ParOptions.Body`) |
| `pre` | `int` | Минимальный известный префикс | `minPrefix` |
| `suf` | `int` | Минимальный известный суффикс | `minSuffix` |

> **Примечание:** В оригинале все индексы — это указатели на `char` (байты).
> В xpar вместо указателей используются индексы в массиве графем,
> полученном через `GetGraphemes(line)`. Это избавляет от необходимости
> вычислять char-позиции через `StringInfo` на каждой итерации.

### 5. Вспомогательные методы (private, в LineReader)

```csharp
/// <summary>
/// Получить список графем-кластеров строки.
/// </summary>
private static IReadOnlyList<string> GetGraphemes(string line)

/// <summary>
/// Подсчитать количество графем в строке.
/// </summary>
private static int CountGraphemes(string line)
```

> **Примечание:** Вместо работы с char-индексами (как в оригинале),
> xpar будет работать с массивами графем. Это упрощает алгоритм —
> не нужно вычислять char-позиции, а индекс в массиве = номер графема.
> Возвращаемое значение (comprelen/comsuflen) — это индекс в массиве графем.

### 6. Тесты (в xParTests, класс CompresuflenTests)

**Базовые тесты:**
1. Одна строка — префикс = вся строка (без body), суффикс = 0
2. Две одинаковые строки — префикс = вся строка, суффикс = 0
3. Две строки с общим префиксом «abc» — comprelen = 3
4. Две строки с общим суффиксом «xyz» — comsuflen = 3

**Тесты с body=0 (по умолчанию):**
5. Строки «> text1 .» и «> text2 .» — префикс остановится перед body-символами
6. Строки с body-символами в середине — суффикс остановится перед trailing body

**Тесты с body=1:**
7. Строки «abc def» и «abc xyz» — body=1, префикс до последнего non-space non-body
8. Строки «abc  » и «abc » — body=1, суффикс с корректировкой пробела

**Тесты с pre/suf > 0:**
9.已知 префикс >= 2 — начинаем сравнение с 3-го графема
10.已知 суффикс >= 1 — начинаем сравнение, отступив 1 графем от конца

**Unicode-тесты:**
11. Строки с эмодзи «👋🌍» — графем-кластеры корректно считаются
12. Строки с composing-символами «é» (e + combining acute) — один графем

### 7. Интеграция

Метод `Compresuflen` добавляется в класс `LineReader` как `public static`.
На данном шаге он не вызывается из `Transform` — это будет сделано позже,
когда будет портирована функция `delimit` и основной цикл обработки абзацев.
