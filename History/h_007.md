# Шаг 7: Реализация MarkSuperf — разметка избыточных (superfluous) строк

## Цель шага

Реализовать аналог `marksuperf()` из оригинального `par` — метод, который определяет
и помечает избыточные (superfluous) строки в сегменте. Избыточные строки — это vacant
(пустые бестелесные) и blank строки, которые могут быть отброшены при включённой опции `expel`.

## План шага

### 1. Теоретическая основа (из par.doc)

**superfluous line:**
> Only blank and vacant lines may be superfluous. If contiguous vacant lines lie at the
> beginning or end of a segment, they are all superfluous. But if they lie between two
> non-vacant lines within a segment, then all are superfluous except one--the one which
> contains the fewest non-spaces. In case of a tie, the first of the tied lines is chosen.
> Similarly, if contiguous blank lines lie outside of any segments at the beginning or end
> of the input, they are all superfluous. But if they lie between two segments and/or
> protected lines, then all are superfluous except the first.

> L_BODILESS bits must already be set.

### 2. Ключевые различия (оригинал vs xpar)

| Аспект | Оригинал (par.c) | xpar |
|--------|-------------------|------|
| Входные данные | Два массива: `lines[]` и `props[]` | Единый `IReadOnlyList<LineSegment>` |
| Границы | `endline` — exclusive (за последней строкой) | `endIndex` — inclusive (последняя строка) |
| Подсчёт non-space | Цикл по `char *p`, `*p != ' '` | Перебор графем, подсчёт не-пробельных |
| `mprop` | Указатель на `lineprop` | Индекс лучшего сегмента |

### 3. Соответствие переменных оригинала и xpar

| Переменная в par.c | Тип | Назначение | Переменная / выражение в xpar |
|---|---|---|---|
| `lines` | `const char * const *` | Массив строк | `segments[i].Line` |
| `endline` | `const char * const *` | Указатель за последнюю строку (exclusive) | `endIndex` (inclusive) |
| `props` | `lineprop *` | Массив свойств | `segments[i].Prop` |
| `line` | `const char * const *` | Итератор по строкам | `lineIdx` — индекс в segments |
| `prop` | `lineprop *` | Итератор по свойствам | `segments[lineIdx].Prop` |
| `p` | `const char*` | Указатель для подсчёта non-space | Итерация по графемам |
| `mprop` | `lineprop*` | Указатель на лучший vacant | `bestIdx` — индекс лучшего сегмента |
| `dummy` | `lineprop` | Фиктивный объект для начального mprop | Не нужен (используем `bestIdx = -1`) |
| `inbody` | `int` | Флаг: находимся ли в «теле» (не-vacant) | `inBody` (bool) |
| `num` | `int` | Кол-во non-space в текущей vacant строке | `nonSpaceCount` |
| `mnum` | `int` | Минимальное кол-во non-space | `minNonSpace` |

### 4. Сигнатура метода

```csharp
/// <summary>
/// Размечает избыточные (superfluous) строки в сегменте.
/// Аналог marksuperf() из par.c (строки 631–660).
/// </summary>
/// <param name="segments">Массив сегментов строк (LineSegment)</param>
/// <param name="startIndex">Индекс первого сегмента для обработки (включительно)</param>
/// <param name="endIndex">Индекс последнего сегмента для обработки (включительно)</param>
public static void MarkSuperf(
    IReadOnlyList<LineSegment> segments,
    int startIndex,
    int endIndex)
```

> **Примечание:** Метод не принимает дополнительных параметров,
> т.к. работает только с уже размеченными свойствами (L_BODILESS, L_SUPERF).
> Требует, чтобы L_BODILESS был уже установлен (после вызова `Delimit`).

### 5. Алгоритм (пошагово, адаптация par.c:631–660)

#### Шаг 1. Предварительная разметка всех vacant как SUPERF (par.c 643–645)

```
for lineIdx = startIndex to endIndex:
    if IsVacant(segments[lineIdx].Prop):
        segments[lineIdx].Prop.Flags |= LineFlags.Superf
```

#### Шаг 2. Определение лучшей vacant в каждой группе (par.c 647–658)

```
inBody = false
minNonSpace = 0
bestIdx = -1  // вместо mprop = &dummy

for lineIdx = startIndex to endIndex:
    if IsVacant(segments[lineIdx].Prop):
        // Подсчёт non-space графем в строке
        nonSpaceCount = CountNonSpaceGraphemes(segments[lineIdx].Line)

        if inBody || nonSpaceCount < minNonSpace:
            minNonSpace = nonSpaceCount
            bestIdx = lineIdx
        inBody = false
    else:
        if !inBody:
            // Снять SUPERF с лучшей vacant из предыдущей группы
            if bestIdx >= 0:
                segments[bestIdx].Prop.Flags &= ~LineFlags.Superf
        inBody = true
```

> **Пояснение:** В оригинале `mprop` инициализируется `&dummy`, а потом
> `mprop->flags &= ~L_SUPERF` безопасно выполняется (dummy не используется).
> В xpar используем `bestIdx = -1` для обозначения отсутствия лучшего кандидата.

### 6. Вспомогательные методы (private, в LineReader)

```csharp
/// <summary>
/// Проверить: сегмент помечен как vacant (бестелесный + rc = " ").
/// </summary>
private static bool IsVacant(LineProp prop)
{
    return IsBodiless(prop) && prop.Rc == " ";
}

/// <summary>
/// Подсчитать количество не-пробельных графем в строке.
/// </summary>
private static int CountNonSpaceGraphemes(string line)
```

### 7. Тесты (в xParTests, класс MarkSuperfTests)

**Базовые тесты:**
1. Пустой диапазон → ничего не меняется
2. Один vacant сегмент → Superf = true (vacant в начале/конце → все избыточные)
3. Один non-vacant сегмент → Superf не затрагивается

**Тесты vacant в начале/конце:**
4. Vacant в начале сегмента → Superf = true
5. Vacant в конце сегмента → Superf = true
6. Несколько contiguous vacant в начале → все Superf = true

**Тесты vacant между non-vacant:**
7. Одна vacant между двумя non-vacant → Superf = false (единственная в группе)
8. Две vacant между non-vacant: одна с меньшим non-space → лучшая не Superf
9. Две vacant с равным non-space: первая выбирается (tie-break)

**Тесты нескольких групп:**
10. non-vacant, vacant, vacant, non-vacant, vacant, non-vacant →
    первая группа vacant: лучшая не Superf, одиночная vacant в конце → Superf = true

**Тесты non-vacant не являются vacant:**
11. Bodiless но не vacant (rc != " ") → Superf не затрагивается
12. Обычные строки без Bodiless → Superf не затрагивается

### 8. Интеграция

Метод `MarkSuperf` добавляется в `LineReader` как `public static`.
Вызывается после `Delimit`, т.к. требует установленного `L_BODILESS`.
