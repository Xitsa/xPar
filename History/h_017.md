воспроизводящую логику главного цикла `main()` из par.c (строки 770–930).

### Главный цикл оригинала (par.c 770–930)

```c
for (sawnonblank = oweblank = 0;  ;  ) {
    /* 1. Чтение абзаца */
    inlines = readlines(&props, ...);
    if (endline == inlines) break;  // EOF

    /* 2. Delimit */
    delimit(inlines, endline, bodychars, repeat, body, div, 0, 0, props);

    /* 3. MarkSuperf */
    if (expel)
        marksuperf(inlines, endline, props);

    /* 4. Цикл по блокам */
    firstline = inlines, firstprop = props;
    do {
        if (isbodiless(firstprop)) {
            /* Обработка bodiless строки */
            if (!(invis && isinserted) && !(expel && issuperf)) {
                /* Вывод: обрезка пробелов или repeat-символы */
            }
            ++firstline, ++firstprop;
            continue;
        }

        /* Найти конец блока */
        for (nextline = firstline + 1, nextprop = firstprop + 1;
             nextline < endline && !isbodiless(nextprop) && !isfirst(nextprop);
             ++nextline, ++nextprop);

        /* Вычислить аффиксы */
        setaffixes(firstline, nextline, firstprop, ...);

        /* Форматирование */
        outlines = reformat(firstline, nextline, afp, fs, hang, prefix, suffix,
                            width, cap, fit, guess, just, last, Report, touch,
                            terminalchars, errmsg);

        for (line = outlines; *line; ++line)
            puts(*line);

        firstline = nextline, firstprop = nextprop;
    } while (firstline < endline);
}
```

## Алгоритм для xpar

### Шаг 1: Разбиение на абзацы

В оригинале `readlines()` читает строки по одной из stdin. В xpar все строки
уже загружены. Нужно разбить их на абзацы:

- **Protected line** — первый графем в `options.ProtectChars` → абзац из одной строки, копируется как есть.
- **Blank line** — пустая или только пробелы → пустая строка в выводе.
- **Обычный абзац** — строки между blank/protected/началом/концом.

### Шаг 2: Обработка обычного абзаца

1. Создать `LineSegment[]` из строк абзаца.
2. `LineReader.Delimit(...)` — определение bodiless строк.
3. `LineReader.MarkSuperf(...)` — если `options.Expel`.
4. Цикл по блокам (от `blockStart` до `blockEnd`):
   - **Bodiless строка**:
     - Если `!options.Invis || !isInserted` и `!options.Expel || !isSuperf`:
       - Если `repeat == 0` или `(rc == " " && suffix == 0)`:
         - Обрезать trailing пробелы, добавить в результат.
       - Иначе:
         - Префикс + repeat-символы × n + суффикс, где `n = width - p - s`.
   - **Обычный блок** (от `blockStart` до `blockEnd`, где `blockEnd` — следующий bodiless/first или конец):
     - `LineReader.SetAffixes(...)` → `affixes`
     - `ReformatModule.Reformat(blockSegments, 0, blockLength-1, options, affixes)` → добавить в результат.

### Шаг 3: Определение границ блока

В оригинале:
```c
for (nextline = firstline + 1, nextprop = firstprop + 1;
     nextline < endline && !isbodiless(nextprop) && !isfirst(nextprop);
     ++nextline, ++nextprop);
```

Блок заканчивается перед:
- Bodiless строкой
- Строкой с флагом `First` (начало нового абзаца внутри сегмента, при `div=true`)
- Концом абзаца

### Главный цикл (main loop)

| `nextprop` | `lineprop*` | Свойства следующей строки | `segments[blockEnd].Prop` |
| `outlines` | `char**` | Массив выходных строк | `List<string>` — результат `Reformat` |

| `suffixbak` | `int` | Сохранённое значение suffix | `options.Suffix` |

### Как вычисляются границы блоков

**В оригинале** (par.c 880–883):
```c
for (nextline = firstline + 1, nextprop = firstprop + 1;
     nextline < endline && !isbodiless(nextprop) && !isfirst(nextprop);
     ++nextline, ++nextprop);
```

Цикл идёт **вперёд** от `firstline + 1`, пока не встретит:
- `endline` — конец абзаца
- `isbodiless(nextprop)` — bodiless строка
- `isfirst(nextprop)` — строка с флагом First (новый абзац при div=true)

Результат: `nextline` указывает на строку, **начинающую следующий блок** (exclusive).

**В xpar**:
```csharp
int blockEnd = blockStart + 1;
while (blockEnd <= paraEnd
       && !segments[blockEnd].Prop.Flags.HasFlag(LineFlags.Bodiless)
       && !segments[blockEnd].Prop.Flags.HasFlag(LineFlags.First))
{
    blockEnd++;
}
// blockEnd — индекс первой строки следующего блока (или paraEnd + 1)
// Блок: segments[blockStart .. blockEnd - 1]
```

**Ключевое отличие:** в оригинале `firstline`/`nextline` — это **указатели** на строки,
которые инкрементируются. В xpar — **индексы** в массиве `segments`.
Диапазон блока передаётся в методы как `startIndex` / `endIndex` (оба inclusive).

### Передача сегментов в методы

Все методы (`Delimit`, `MarkSuperf`, `SetAffixes`, `Reformat`) принимают:
- `IReadOnlyList<LineSegment> segments` — **весь** массив сегментов абзаца
- `int startIndex` — индекс начала блока (inclusive)
- `int endIndex` — индекс конца блока (inclusive)

Это отличается от оригинала, где передаются **указатели** на начало и конец:
```c
delimit(firstline, nextline, ...);  // firstline — начало, nextline — за концом
```

В xpar `nextline` (exclusive) преобразуется в `endIndex = blockEnd - 1` (inclusive):
```csharp
// Оригинал: delimit(firstline, nextline, ...)
// xpar:
Delimit(segments, blockStart, blockEnd - 1, ...);
```

### 17.4 Helper для тестов: `ParseOptions`

Для удобства написания тестов добавить вспомогательный метод, который создаёт
`ParOptions` из строк в том виде, в каком их передаёт пользователь в командной строке:

```csharp
/// <summary>
/// Создаёт ParOptions из набора аргументов в пользовательском формате.
/// </summary>
/// <param name="args">Аргументы как в командной строке, например: "w20", "j", "r3".</param>
/// <returns>Готовый ParOptions с инициализированными charset'ами.</returns>
private static ParOptions ParseOptions(params string[] args)
{
    var bodyChars = Charset.Parse("");
    var protectChars = Charset.Parse("");
    var quoteChars = Charset.Parse("> ");
    var whiteChars = Charset.Parse(" \f\n\r\t\v");
    var terminalChars = Charset.Parse(".?!:");

    return ParOptions.Parse(args, bodyChars, protectChars, quoteChars, whiteChars, terminalChars);
}
```

**Примеры использования в тестах:**

```csharp
// Ширина 20, justification
var opts = ParseOptions("w20", "j");

// Repeat=3, guess, quote
var opts = ParseOptions("r3", "g", "q");

// Только значения по умолчанию
var opts = ParseOptions();

// Переопределение: последнее значение побеждает
var opts = ParseOptions("w72", "w20");  // Width = 20
```
