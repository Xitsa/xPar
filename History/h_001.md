# Шаг 1: Создание инфраструктуры параметров

## Цель шага

Создать класс `ParOptions` для хранения всех параметров, соответствующих оригинальному `par` (Par 1.53.0), и добавить статический метод для разбора аргументов командной строки.

## План шага

1. Создать класс `ParOptions` в проекте xParLib со следующими свойствами:

   **Целочисленные параметры:**
   - `Hang` (int) — пропустить первые N строк абзаца при анализе префиксов/суффиксов (по умолч. 0)
   - `Prefix` (int?) — длина префикса (null = вычислять автоматически)
   - `Repeat` (int) — режим обработки бестелесных строк (по умолч. 0)
   - `Suffix` (int?) — длина суффикса (null = вычислять автоматически)
   - `Tab` (int) — ширина табуляции (по умолч. 1)
   - `Width` (int) — максимальная длина строки вывода (по умолч. 72)

   **Булевы параметры:**
   - `Body` (bool) — учитывать символы тела в префиксах/суффиксах (по умолч. false)
   - `Cap` (bool) — считать все слова капитализированными (по умолч. false)
   - `Div` (bool) — использовать отступы как разделители абзацев (по умолч. false)
   - `Err` (bool) — отправлять сообщения в stderr (по умолч. false)
   - `Expel` (bool) — отбрасывать лишние строки (по умолч. false)
   - `Fit` (bool) — сузить абзац для лучшего соответствия (по умолч. false)
   - `Guess` (bool) — сохранять широкие разрывы предложений (по умолч. false)
   - `Invis` (bool) — скрывать строки, вставленные quote (по умолч. false)
   - `Just` (bool) — выравнивать абзацы (по умолч. false)
   - `Last` (bool) — обрабатывать последние строки как остальные (по умолч. false)
   - `Quote` (bool) — добавлять пустые строки между уровнями цитирования (по умолч. false)
   - `Report` (bool) — сообщать об ошибках для слишком длинных слов (по умолч. false)
   - `Touch` (bool?) — перемещать суффиксы влево (null = вычислять как fit || last)

   **Параметры справки:**
   - `Help` (bool) — запрошена справка
   - `Version` (bool) — запрошена версия

2. Добавить статический метод `Parse(string[] args)`:
   - Поддержка форматов: `h<hang>`, `p<prefix>`, `r<repeat>`, `s<suffix>`, `T<Tab>`, `w<width>`
   - Поддержка булевых флагов: `b`, `c`, `d`, `E`, `e`, `f`, `g`, `i`, `j`, `l`, `q`, `R`, `t`
   - Поддержка `help` и `version`
   - Автоматическое определение типа параметра по числу (<=8 → prefix, >8 → width)
   - Возвращает экземпляр `ParOptions` с заполненными параметрами

   **Вычисление зависимых значений по умолчанию:**
   - `Touch` вычисляется внутри `Parse` как `Fit || Last` (не зависит от входных строк)
   - `Prefix` и `Suffix` НЕ вычисляются в `Parse` — они зависят от входных строк (comprelen/comsuflen)
   - Вычисление `Prefix` на основе `Hang`, `Quote` и входных строк будет выполнено в `Transform`
   - Если параметр задан несколько раз — используется последнее значение

3. Добавить тесты в xParTests:
   - Проверка разбора целочисленных параметров (`h1`, `p5`, `r3`, `s2`, `T8`, `w72`)
   - Проверка разбора булевых параметров (`b`, `c`, `d`, `f`, `g`, `j`, `l`, `q`)
   - Проверка специальных случаев (`help`, `version`, числа без буквы)
   - Проверка значений по умолчанию
   - Проверка правила "последнее значение побеждает"
   - Проверка порядка вычисления зависимых параметров (`hang` перед `prefix`, `fit`/`last` перед `touch`)

4. Обновить документацию:
   - Добавить описание класса `ParOptions` в README.md
   - Обновить QWEN.md
