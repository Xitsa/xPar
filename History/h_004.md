# Шаг 4: Реализация ReadLines — чтение и аннотирование строк

## Цель шага

Реализовать аналог `readlines()` из оригинального `par` — метод, который принимает массив строк,
обрабатывает их с учётом защитных символов, пустых строк, пробелов, табуляции и quote-вставок,
и возвращает структурированный результат с массивом `LineProp` и позицией следующей необработанной строки.

## План шага

### 1. Создать структуру `LineProp`

Аналог `struct lineprop` из par.c, но с `Rc` как строкой (графем-кластер):

```csharp
[Flags]
public enum LineFlags
{
    None = 0,
    Bodiless = 1,    // Бестелесная строка
    Inserted = 2,    // Вставлена механизмом quote
    First = 4,       // Первая строка абзаца
    Superf = 8,      // Избыточная строка
}

public struct LineProp
{
    public int P { get; set; }           // Длина префикса (или fallback prelen)
    public int S { get; set; }           // Длина суффикса (или fallback suflen)
    public LineFlags Flags { get; set; } // Флаги
    public string Rc { get; set; }       // Повторяющийся графем-кластер бестелесной строки
}
```

> **Примечание:** В оригинальном `par` поле `rc` имеет тип `char` (1 байт).
> В xpar это `string`, т.к. повторяющийся элемент может быть графем-кластером
> из нескольких Unicode code points (например `👨‍👩‍👧‍👦` или `é`).

### 2. Создать класс `LineSegment` (пара строка + свойства)

```csharp
public class LineSegment
{
    public string Line { get; set; }
    public LineProp Prop { get; set; }

    public LineSegment(string line, LineProp prop)
    {
        Line = line;
        Prop = prop;
    }
}
```

### 3. Создать класс `ReadLinesResult`

Результат работы метода — массив пар + позиция:

```csharp
public class ReadLinesResult
{
    public LineSegment[] Segments { get; }   // Массив пар (строка, свойства)
    public int NextIndex { get; }            // Позиция следующей необработанной строки
    public bool IsEof { get; }               // Достигнут ли конец входных данных

    public ReadLinesResult(LineSegment[] segments, int nextIndex, bool isEof);
}
```

> **Примечание:** Пара `LineSegment` добавляется целиком, что исключает
> рассогласование размеров массивов при портировании логики алгоритма.

### 4. Создать класс `LineReader`

С методом `ReadLines`:

```csharp
public class LineReader
{
    public ReadLinesResult ReadLines(
        IReadOnlyList<string> inputLines,
        int startIndex,
        Charset protectChars,
        Charset quoteChars,
        Charset whiteChars,
        int tab,
        bool invis,
        bool quote);
}
```

**Логика (аналог readlines из par.c):**

- Читает строки из `inputLines` начиная с `startIndex`
- Останавливается при:
  1. **EOF** — конец массива строк
  2. **Protected line** — первый графем-кластер строки — защитный символ (строка НЕ включается, позиция указывает на неё)
  3. **Blank line** — пустая строка или строка из одних пробелов (строка НЕ включается, позиция указывает на неё)
- Для каждой строки обработка идёт **по графем-кластерам** (как в текущем `Transform`):
  - Используется `StringInfo.GetTextElementEnumerator` для перебора графем
  - NUL-символы пропускаются
  - whitespace-графемы (кроме `\n`) заменяются на пробел `" "`
  - Табуляция расширяется до пробелов (с шагом `tab`), считается ширина каждого графема
  - Если `quote` = true: вставляются vacant-строки между уровнями цитирования
  - Создаётся `LineProp` для каждой строки (flags: `Inserted` для вставленных quote)
- Возвращает `ReadLinesResult` с массивом `LineSegment[]` и `NextIndex`

### 4. Нюансы quote-вставки (из par.c)

Если `quote` = true:
- Для каждой пары соседних строк сравниваются quoteprefix'ы (максимальная строка quote-символов в начале)
- Если quoteprefix'ы разные:
  - Если `invis` = false И одна из строк состоит только из quote-символов и пробелов → укоротить эту строку до общего префикса
  - Иначе → вставить новую строку с общим префиксом двух quoteprefix'ов (с флагом `Inserted`)

### 5. Добавить тесты в xParTests

**Тесты для ReadLines:**
- Пустой вход → пустой результат
- Обычные строки → все включены, NextIndex = EOF
- Строка с защитным символом в середине → останавливается, NextIndex указывает на protected line
- Пустая строка в середине → останавливается, NextIndex указывает на blank line
- Tab expansion: `"a\tb"` с tab=4 → `"a   b"`
- whitespace замена: `"a\fb"` → `"a b"`
- quote = true: вставка vacant lines между разными уровнями цитирования
- quote + invis: укорачивание строк вместо вставки

### 6. Интегрировать в StringTransformer

`Transform` пока вызывает `ReadLines` для получения первого блока строк,
но дальнейшую обработку (comprelen/comsuflen, word wrapping) пока не делает —
это будет на следующем шаге.

### 7. Алгоритм ReadLines (соответствие readlines из par.c)

**Исходный метод:** `readlines()` из `par.c`, строки 314–478.

#### 7.1. Переменные и их замены

| Переменная в par.c | Тип | Назначение | Замена в xpar |
|---|---|---|---|
| `cbuf` | `buffer*` | Буфер посимвольного накопления строки | Не нужен — строки уже готовы |
| `lbuf` | `buffer*` | Буфер указателей на строки | `List<LineSegment>` |
| `lpbuf` | `buffer*` | Буфер `lineprop` | Часть `List<LineSegment>` |
| `ch` | `char` | Текущий символ из `getchar()` | `string grapheme` — графем-кластер |
| `ln` | `char*` | Готовая строка (копия из cbuf) | `string currentLine` |
| `empty` | `int` | Строка пока пуста (0 символов) | `bool isEmpty` |
| `blank` | `int` | Строка — только пробелы | `bool isBlank` |
| `firstline` | `int` | Первая строка ввода | `bool isFirstLine` |
| `qsonly` | `int` | Строка — только quote + пробелы | `bool isQuoteOnly` |
| `oldqsonly` | `int` | Предыдущая — только quote | `bool prevQuoteOnly` |
| `qpend` | `char*` | Конец quote-префикса текущей | `int quotePrefixEnd` (индекс) |
| `oldln`, `oldqpend` | `char*` | Предыдущая строка, конец quote | `string prevLine`, `int prevQuotePrefixEnd` |
| `vln` | `char*` | Вставляемая vacant-строка | `string vacantLine` |
| `vprop` | `lineprop` | Свойства обычной строки (нули) | `new LineProp()` |
| `iprop` | `lineprop` | Свойства вставленной (L_INSERTED) | `new LineProp { Flags = LineFlags.Inserted }` |
| `lines`, `pprops` | `char**`, `lineprop**` | Возвращаемые массивы | `LineSegment[] Segments` |

#### 7.2. Почему не нужны буферы (cbuf, lbuf, lpbuf)

В `par` буферы нужны из-за посимвольного чтения `getchar()`:
- `cbuf` — накапливает символы до `\n`
- `lbuf`/`lpbuf` — накапливают строки и props

В xpar **все строки уже в памяти** (`IReadOnlyList<string>`):
- Нет `cbuf` — строка готова, обрабатывается целиком (tab expansion, whitespace)
- Нет `lbuf`/`lpbuf` — `List<LineSegment>` добавляет пару целиком
- Нет `NULL`-терминатора — `IsEof` + `Segments.Length`

#### 7.3. Пошаговый алгоритм

**Шаг 1. Инициализация** (par.c 338–354)
```
segments = new List<LineSegment>()
vprop = new LineProp()
iprop = new LineProp { Flags = LineFlags.Inserted }
isEmpty = isBlank = isFirstLine = true
prevLine = "", prevQuotePrefixEnd = 0, prevQuoteOnly = false
```

**Шаг 2. Цикл по входным строкам** (par.c 356–445)

```
for i = startIndex to inputLines.Count - 1:
    currentLine = inputLines[i]
    processedLine = ProcessLine(currentLine, protectChars, whiteChars, tab)
    
    if processedLine.IsProtected(protectChars):
        return ReadLinesResult(segments, i, isEof: false)
    
    if quote:
        ProcessQuoteLogic(...)
    
    segments.Add(new LineSegment(processedLine, vprop))
    isBlank = processedLine.IsBlank()
    isFirstLine = false

// EOF — все строки прочитаны
return ReadLinesResult(segments, inputLines.Count, isEof: true)
```

**Шаг 2a. ProcessLine — обработка одной строки по графемам**

```
processedLine = ""
currentWidth = 0

enumerator = StringInfo.GetTextElementEnumerator(currentLine)
while enumerator.MoveNext():
    grapheme = enumerator.GetTextElement()
    
    if grapheme == "\0": continue  // NUL — пропуск
    
    if grapheme == "\t":
        // Tab expansion
        spacesToAdd = tab - (currentWidth % tab)
        processedLine += new string(' ', spacesToAdd)
        currentWidth += spacesToAdd
        continue
    
    if whiteChars.IsMember(grapheme):
        grapheme = " "
    
    processedLine += grapheme
    currentWidth += UnicodeCalculator.GetWidth(grapheme[0])

return processedLine
```

**Проверка protected** (par.c 419–423):
```
// Первый графем-кластер строки — если в protectChars → protected
firstGrapheme = GetFirstGrapheme(line)
return protectChars.IsMember(firstGrapheme)
```

**Шаг 3. Quote-логика** (par.c 368–416)

```
quotePrefixEnd = FindQuotePrefixEnd(processedLine, quoteChars)
isQuoteOnly = IsQuoteOnly(processedLine, quoteChars, quotePrefixEnd)

if !isFirstLine:
    commonPrefix = CommonPrefix(
        processedLine.Substring(0, quotePrefixEnd),
        prevLine.Substring(0, prevQuotePrefixEnd))
    
    if commonPrefix != prevLine[..prevQuotePrefixEnd] ||
       commonPrefix != processedLine[..quotePrefixEnd]:
        
        if !invis && (prevQuoteOnly || isQuoteOnly):
            // Укоротить строку из quote
            if prevQuoteOnly: prevLine = commonPrefix
            if isQuoteOnly: processedLine = commonPrefix
        else:
            // Вставить vacant line
            segments.Add(new LineSegment(commonPrefix, iprop))

prevLine = processedLine
prevQuotePrefixEnd = quotePrefixEnd
prevQuoteOnly = isQuoteOnly
```

**Шаг 4. Blank line проверка** (par.c 363–365)

После обработки строки:
```
if isBlank && !isEmpty:
    // Blank line — стоп, не включаем
    return ReadLinesResult(segments, i, isEof: false)
```

#### 7.4. Вспомогательные методы

```csharp
// Обработать строку: tab expansion, whitespace → пробел, NUL пропуск
private static string ProcessLine(string line, Charset protectChars,
                                  Charset whiteChars, int tab)

// Получить первый графем-кластер строки
private static string GetFirstGrapheme(string line)

// Найти конец quote-префикса (индекс после последнего quote-символа)
private static int FindQuotePrefixEnd(string line, Charset quoteChars)

// Проверить: строка после quotePrefixEnd — только quote + пробелы
```

## Результат шага

- Созданы типы в xParLib:
  - `LineFlags` — флаги свойств строк (Bodiless, Inserted, First, Superf)
  - `LineProp` — свойства строки (P, S, Flags, Rc), где Rc — строка (графем-кластер)
  - `LineSegment` — пара (строка, LineProp) для избежания рассогласования массивов
  - `ReadLinesResult` — результат с массивом `LineSegment[]`, `NextIndex`, `IsEof`
  - `LineReader` — класс с методом `ReadLines` (аналог `readlines()` из par.c)

- Реализована логика `ReadLines`:
  - Остановка по EOF, protected line, blank line
  - `ProcessLine` — обработка по графем-кластерам (tab expansion, whitespace → пробел, NUL пропуск)
  - Quote-логика — вставка vacant lines между разными уровнями цитирования
