# Шаг 6: Реализация Delimit — определение бестелесных строк и разметка префиксов/суффиксов

## Цель шага

Реализовать аналог `delimit()` из оригинального `par` — метод, который принимает сегменты строк
с их свойствами, определяет бестелесные строки (bodiless lines), вычисляет для них префиксы/суффиксы
и рекурсивно обрабатывает подблоки между бестелесными строками.

## План шага

### 1. Теоретическая основа (из par.doc)

**bodiless line:**
> A line which, if stripped of its first <p> and last <s> characters, would be blank
> (or, if the line was not inserted by the <quote> feature and the parameter <repeat>
> is non-zero, would consist of the same character repeated at least <repeat> times).

**Алгоритм delimit (из par.doc):**
> Sets fields in each lineprop in the parallel array props as appropriate,
> except for the L_SUPERF flag, which is never set. It is assumed that the
> comprelen and comsuflen of the lines in lines have already been determined
> to be at least pre and suf, respectively.

### 2. Ключевые различия (оригинал vs xpar)

| Аспект | Оригинал (par.c) | xpar |
|--------|-------------------|------|
| Входные данные | Два параллельных массива: `lines[]` и `props[]` | Единый `IReadOnlyList<LineSegment>` (строка + свойства) |
| Границы | `endline` — указатель ЗА последнюю строку (exclusive) | `endIndex` — индекс последней строки (inclusive) |
| Повторяющийся символ | `char rc` (1 байт) | `string Rc` (графем-кластер) |
| Проверка «повторяемости» | Посимвольное сравнение `*p != rc` | Сравнение графем-кластеров |
| Расстояние между указателями | `end - p` (разница байт) | Подсчёт графем между позициями |

### 3. Соответствие переменных оригинала и xpar

| Переменная в par.c | Тип | Назначение | Переменная / выражение в xpar |
|---|---|---|---|
| `lines` | `const char * const *` | Массив строк | `segments[i].Line` |
| `endline` | `const char * const *` | Указатель за последнюю строку (exclusive) | `endIndex` (inclusive) — индекс последней строки |
| `props` | `lineprop *` | Параллельный массив свойств | `segments[i].Prop` (мутируемый) |
| `bodychars` | `charset*` | Набор body-символов | `bodyChars` (параметр `Charset`) |
| `repeat` | `int` | Минимальное кол-во повторений для bodiless | `repeat` (параметр `int`, соответствует `ParOptions.Repeat`) |
| `body` | `int` | Флаг режима body (0 или 1) | `body` (параметр `bool`, соответствует `ParOptions.Body`) |
| `div` | `int` | Флаг деления на абзацы по отступам | `div` (параметр `bool`, соответствует `ParOptions.Div`) |

| `anybodiless` | `int` | Есть ли бестелесные строки | `hasAnyBodiless` |
| `status` | `int` | Статус первой строки для div | `firstStatus` |

### 4. Сигнатура метода

```csharp
/// <summary>
/// Определяет бестелесные строки и размечает свойства сегментов.
/// Аналог delimit() из par.c (строки 544–628).
/// </summary>
/// <param name="segments">Массив сегментов строк (LineSegment)</param>
/// <param name="startIndex">Индекс первого сегмента для обработки (включительно)</param>
/// <param name="endIndex">Индекс последнего сегмента для обработки (включительно)</param>
/// <param name="bodyChars">Набор body-символов</param>
/// <param name="repeat">Минимальное кол-во повторений для bodiless (par: r/repeat)</param>
/// <param name="body">Флаг режима body (false = 0, true = 1)</param>
/// <param name="div">Флаг деления по отступам (false = 0, true = 1)</param>
/// <param name="minPrefix">Минимальная известная длина префикса (в графемах)</param>
/// <param name="minSuffix">Минимальная известная длина суффикса (в графемах)</param>
public static void Delimit(
    IReadOnlyList<LineSegment> segments,
    int startIndex,
    int endIndex,
    Charset bodyChars,
    int repeat,
    bool body,
    bool div,
    int minPrefix,
    int minSuffix)
```

> **Примечание:** `endIndex` — inclusive (в оригинале `endline` — exclusive).
> При вызове `Compresuflen` внутри `Delimit` нужно передавать те же `startIndex` и `endIndex`.

### 5. Алгоритм (пошагово, адаптация par.c:544–628)

#### Шаг 1. Базовые случаи (par.c 562–568)

```
if startIndex > endIndex: return  // endline == lines

if startIndex == endIndex:  // endline == lines + 1
    segments[startIndex].Prop.Flags |= LineFlags.First
    segments[startIndex].Prop.P = minPrefix
    segments[startIndex].Prop.S = minSuffix
    return
```

#### Шаг 2. Вызов Compresuflen (par.c 570)

```
result = Compresuflen(segments, startIndex, endIndex, bodyChars, body, minPrefix, minSuffix)
minPrefix = result.Prefix
minSuffix = result.Suffix
```

#### Шаг 3. Определение bodiless строк (par.c 572–594)

Для каждого сегмента в диапазоне `startIndex .. endIndex`:

```
for lineIdx = startIndex to endIndex:
    segments[lineIdx].Prop.Flags |= LineFlags.Bodiless
    segments[lineIdx].Prop.P = minPrefix
    segments[lineIdx].Prop.S = minSuffix

    totalGraphemes = CountGraphemes(segments[lineIdx].Line)
    bodyEnd = totalGraphemes - minSuffix  // конец тела (exclusive)
    bodyStart = minPrefix                   // начало тела (exclusive)

    if bodyStart < bodyEnd:
        repeatGrapheme = графем на позиции bodyStart
    else:
        repeatGrapheme = " "

    // Проверка: не bodiless, если rc != ' ' И (вставлена ИЛИ !repeat ИЛИ тело < repeat)
    if repeatGrapheme != " " &&
       (IsInserted(segments[lineIdx].Prop) || repeat == 0 ||
        (bodyEnd - bodyStart) < repeat):
        segments[lineIdx].Prop.Flags &= ~LineFlags.Bodiless
    else:
        // Проверить: все графемы в теле == repeatGrapheme
        for g = bodyStart + 1 to bodyEnd - 1:
            if графем(g) != repeatGrapheme:
                segments[lineIdx].Prop.Flags &= ~LineFlags.Bodiless
                break

    if IsBodiless(segments[lineIdx].Prop):
        hasAnyBodiless = true
        segments[lineIdx].Prop.Rc = repeatGrapheme
```

#### Шаг 4. Рекурсивная обработка (par.c 596–613)

Если есть bodiless строки:

```
if hasAnyBodiless:
    lineIdx = startIndex
    while lineIdx <= endIndex:
        if IsBodiless(segments[lineIdx].Prop):
            lineIdx++
            continue

        // Найти следующий bodiless
        nextLineIdx = lineIdx + 1
        while nextLineIdx <= endIndex && !IsBodiless(segments[nextLineIdx].Prop):
            nextLineIdx++

        // Рекурсивный вызов для подблока [lineIdx .. nextLineIdx-1]
        // nextLineIdx — первый bodiless, значит endIndex подблока = nextLineIdx - 1
        if lineIdx <= nextLineIdx - 1:
            Delimit(segments, lineIdx, nextLineIdx - 1, bodyChars, repeat, body, div, minPrefix, minSuffix)

        lineIdx = nextLineIdx
    return
```

#### Шаг 5. Обработка div (par.c 615–628)

Если нет bodiless строк и `div == false`:

```
if !div:
    segments[startIndex].Prop.Flags |= LineFlags.First
    return
```

Если `div == true`:

```
firstStatus = (графем minPrefix в segments[startIndex].Line) == " "
for lineIdx = startIndex to endIndex:
    currentStatus = (графем minPrefix в segments[lineIdx].Line) == " "
    if currentStatus == firstStatus:
        segments[lineIdx].Prop.Flags |= LineFlags.First
```

### 7. Тесты (в xParTests, класс DelimitTests)

**Базовые тесты:**
1. Пустой диапазон → ничего не меняется
2. Один сегмент → First = true, P/S = minPrefix/minSuffix
3. Два одинаковых сегмента → Compresuflen вычисляет префикс/суффикс, оба становятся First

**Тесты bodiless:**
4. Строка из одних пробелов между двумя строками → bodiless, Rc = " "
5. Строка из повторяющихся символов `---` (repeat=3) → bodiless, Rc = "-"
6. Строка из повторяющихся символов `--` (repeat=3) → НЕ bodiless (слишком короткая)
7. Вставленная строка (Inserted) с rc != " " → НЕ bodiless

**Тесты рекурсии:**
8. Bodiless строка в середине → подблоки до и после обрабатываются рекурсивно
9. Несколько bodiless строк подряд → каждая размечена, подблоки между ними

**Тесты div:**
10. div=false, нет bodiless → только первая строка получает First
11. div=true, разные отступы → строки с одинаковым статусом пробела получают First

**Тесты body=1:**
12. body=1: префикс/суффикс с учётом body-символов

### 8. Интеграция

Метод `Delimit` добавляется в `LineReader` как `public static`.
На данном шаге он не вызывается из `Transform` — это будет сделано позже,
когда будет портирован основной цикл обработки абзацев (main loop).
