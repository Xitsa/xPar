# Шаг 9: Реализация Reformat — перенос логики форматирования строк

## Цель шага

Реализовать основную логику форматирования абзацев (word wrapping, justification и т.д.) в методе `Reformat`,
согласно опциям `ParOptions` и с учётом `SetAffixesResult`. **На этом шаге создаётся только скелет функции.**

## План шага

### 1. Определение сигнатуры метода `Reformat`

Метод будет иметь следующую сигнатуру:
```csharp
public static string[] Reformat(
    IReadOnlyList<LineSegment> segments,
    int startIndex,
    int endIndex,
    ParOptions options,
    SetAffixesResult affixes)
```
- `segments`: Входные строки абзаца (IP) в виде `LineSegment`, уже обработанные `LineReader`, `Delimit`, `MarkSuperf`.
- `startIndex`, `endIndex`: **Включительные границы** обрабатываемого диапазона сегментов.
- `options`: Параметры форматирования (`Width`, `Fit`, `Just`, `Last`, `Hang` и др.).
- `affixes`: Результат `SetAffixes`, содержащий вычисленные `Prefix`, `Suffix`, `AugmentedFallbackPre`, `FallbackSuf`.

### 2. Реализация основной логики форматирования

- Итерация по входным `segments` абзаца в пределах `startIndex` и `endIndex`.
- Применение `options` (`Width`, `Fit`, `Just`, `Last`) и `affixes` (`Prefix`, `Suffix`) для определения переносов строк и выравнивания.
- Обработка переносов слов (word wrapping) с учётом `options.Width`.
- Выравнивание строк (`options.Just`).
- Учёт `options.Fit` для более плотного заполнения строки.
- Обработка последней строки (`options.Last`).
- Корректная работа с Unicode графем-кластерами.
- Возврат отформатированных строк в виде `string[]`.

### 3. Соответствие параметров `par` и `xpar`

| Параметр `par` | Описание | Соответствие в `xpar` | Тип |
|---|---|---|---|
| `width` | Максимальная длина строки вывода | `options.Width` | `int` |
| `fit` | Сузить абзац для лучшего соответствия | `options.Fit` | `bool` |
| `just` | Выравнивать абзацы | `options.Just` | `bool` |
| `last` | Обрабатывать последние строки как остальные | `options.Last` | `bool` |
| `prefix` | Длина префикса (из `setaffixes`) | `affixes.Prefix` | `int` |
| `suffix` | Длина суффикса (из `setaffixes`) | `affixes.Suffix` | `int` |
| `bodychars` | Набор body-символов | `options.BodyChars` | `Charset` |
| `whitechars` | Набор пробельных символов | `options.WhiteChars` | `Charset` |
| `quotechars` | Набор quote-символов | `options.QuoteChars` | `Charset` |
| `hang` | Пропустить первые N строк при вычислении аффиксов | `options.Hang` | `int` |
| `invis` | Скрывать вставленные строки | `options.Invis` | `bool` |
| `repeat` | Режим обработки бестелесных строк | `options.Repeat` | `int` |
| `div` | Использовать отступы как разделители абзацев | `options.Div` | `bool` |
| `cap` | Считать все слова капитализированными | `options.Cap` | `bool` |
| `err` | Отправлять сообщения в stderr | `options.Err` | `bool` |
| `expel` | Отбрасывать лишние строки | `options.Expel` | `bool` |
| `guess` | Сохранять широкие разрывы предложений | `options.Guess` | `bool` |
| `touch` | Перемещать суффиксы влево | `options.Touch` | `bool?` |

### 4. Добавление тестов

- Тесты на базовое форматирование строк (`Width`).
- Тесты на выравнивание (`Just`).
- Тесты с учётом `Fit` и `Last`.
- Тесты с использованием `Prefix` и `Suffix` из `SetAffixesResult`.
- Тесты с Unicode символами.
- Тесты на граничные случаи (пустой IP, одна строка в IP).

### 5. Интеграция

- Метод `Reformat` будет вызываться из `StringTransformer.Transform` после получения `LineSegment[]` и `SetAffixesResult`.
