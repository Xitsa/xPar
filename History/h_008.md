# Шаг 8: Реализация SetAffixes — вычисление финальных префикса и суффикса IP

## Цель шага

Реализовать аналог `setaffixes()` из оригинального `par` — метод, который принимает
сегменты входного абзаца (IP) и вычисляет итоговые значения префикса и суффикса,
которые будут использоваться при переформатировании.

## План шага

### 1. Теоретическая основа (из par.doc)

**fallback prelen (suflen):**
> The fallback prelen (suflen) of an IP is: the comprelen (comsuflen) of the IP,
> if the IP contains at least two lines; otherwise, the comprelen (comsuflen) of
> the block containing the IP, if the block contains at least two lines; otherwise,
> the length of the longer of the prefixes (suffixes) of the bodiless lines just
> above and below the block, if the segment containing the block has any bodiless
> lines; otherwise, 0.

**augmented fallback prelen:**
> Let <fp> be the fallback prelen of an IP. If the IP contains more than one line,
> or if <quote> is 0, then the augmented fallback prelen of the IP is simply <fp>.
> Otherwise, it is <fp> plus the number of quote characters immediately following
> the first <fp> characters of the line.

**setaffixes (из par.doc):**
> Sets the values of prefix and suffix for an IP. If either is unset (<0),
> it is computed from the comprelen/comsuflen of the IP (excluding the first <hang> lines)
> or from the augmented fallback prelen / fallback suflen.

### 2. Алгоритм setaffixes (par.c:664–696)

Функция решает: если `prefix` или `suffix` ещё не установлены (.HasValue),
вычислить их по формуле:
- Если строк в IP > hang + 1 → взять `comprelen`/`comsuflen` строк начиная с `hang`
- Иначе → взять `augmented fallback prelen` / `fallback suflen`

### 3. Ключевые различия (оригинал vs xpar)

| Аспект | Оригинал (par.c) | xpar |
|--------|-------------------|------|
| Входные данные | Два массива: `inlines[]` и `props[]` | Единый `IReadOnlyList<LineSegment>` |
| Границы | `endline` — exclusive | `endIndex` — inclusive |
| Возврат значений | Указатели `*pafp`, `*pfs`, `*pprefix`, `*psuffix` | Возвращаемый record |
| Подсчёт quote-символов | Цикл `while (*p && csmember(*p, quotechars)) ++p;` | Перебор графем, проверка quote |

### 4. Соответствие переменных оригинала и xpar

| Переменная в par.c | Тип | Назначение | Переменная / выражение в xpar |
|---|---|---|---|
| `inlines` | `const char * const *` | Массив строк IP | `segments[i].Line` |
| `endline` | `const char * const *` | Указатель за последнюю строку (exclusive) | `endIndex` (inclusive) |
| `props` | `const lineprop *` | Массив свойств (только чтение) | `segments[i].Prop` |
| `bodychars` | `charset*` | Набор body-символов | `bodyChars` (параметр `Charset`) |
| `quotechars` | `charset*` | Набор quote-символов | `quoteChars` (параметр `Charset`) |
| `hang` | `int` | Пропуск первых <hang> строк | `hang` (параметр `int`) |
| `body` | `int` | Флаг режима body | `body` (параметр `bool`) |
| `quote` | `int` | Флаг quote | `quote` (параметр `bool`) |
| `*pafp` | `int*` | Augmented fallback prelen | Поле `AugmentedFallbackPre` в result |
| `*pfs` | `int*` | Fallback suflen | Поле `FallbackSuf` в result |
| `*pprefix` | `int*` | Итоговый префикс (вход: <0 = unset) | `int? prefix` (null = unset) |
| `*psuffix` | `int*` | Итоговый суффикс (вход: <0 = unset) | `int? suffix` (null = unset) |
| `numin` | `int` | Количество строк в IP | `endIndex - startIndex + 1` |
| `pre`, `suf` | `int` | Временные comprelen/comsuflen | Локальные переменные |
| `p` | `const char*` | Указатель для подсчёта quote | Индекс графема |

### 5. Тип результата — record `SetAffixesResult`

```csharp
/// <summary>
/// Результат вычисления аффиксов IP.
/// </summary>
public readonly record struct SetAffixesResult(
    int Prefix,               // Итоговый префикс
    int Suffix,               // Итоговый суффикс
    int AugmentedFallbackPre, // Augmented fallback prelen
    int FallbackSuf           // Fallback suflen
);
```

### 6. Сигнатура метода

```csharp
/// <summary>
/// Вычисляет итоговые значения префикса и суффикса для IP.
/// Аналог setaffixes() из par.c (строки 664–696).
/// </summary>
/// <param name="segments">Массив сегментов строк (LineSegment)</param>
/// <param name="startIndex">Индекс первого сегмента IP (включительно)</param>
/// <param name="endIndex">Индекс последнего сегмента IP (включительно)</param>
/// <param name="bodyChars">Набор body-символов</param>
/// <param name="quoteChars">Набор quote-символов</param>
/// <param name="hang">Пропуск первых hang строк при вычислении</param>
/// <param name="body">Флаг режима body (false = 0, true = 1)</param>
/// <param name="quote">Флаг quote (false = 0, true = 1)</param>
/// <param name="prefix">Входной префикс (если HasValue, используется как есть)</param>
/// <param name="suffix">Входной суффикс (если HasValue, используется как есть)</param>
/// <returns>SetAffixesResult с итоговыми значениями</returns>
public static SetAffixesResult SetAffixes(
    IReadOnlyList<LineSegment> segments,
    int startIndex,
    int endIndex,
    Charset bodyChars,
    Charset quoteChars,
    int hang,
    bool body,
    bool quote,
    int? prefix,
    int? suffix)
```

### 7. Алгоритм (пошагово, адаптация par.c:664–696)

#### Шаг 1. Вычисление количества строк (par.c 678)

```
numin = endIndex - startIndex + 1
```

#### Шаг 2. Вызов Compresuflen если нужно (par.c 680-681)

```
computedPrefix = 0, computedSuffix = 0
if (!prefix.HasValue || !suffix.HasValue) && numin > hang + 1:
    csResult = Compresuflen(segments, startIndex + hang, endIndex, bodyChars, body, 0, 0)
    computedPrefix = csResult.Prefix
    computedSuffix = csResult.Suffix
```

#### Шаг 3. Вычисление augmented fallback prelen (par.c 683-688)

```
// Базовый fallback prelen = P первой строки (уже установлен Delimit)
fallbackPre = segments[startIndex].Prop.P

// Augmented: если IP из 1 строки И quote=true, добавляем quote-символы
if numin == 1 && quote:
    graphemes = GetGraphemes(segments[startIndex].Line)
    pos = fallbackPre
    while pos < graphemes.Count && quoteChars.IsMember(graphemes[pos]):
        pos++
    augmentedFallbackPre = pos
else:
    augmentedFallbackPre = fallbackPre

fallbackSuf = segments[startIndex].Prop.S
```

#### Шаг 4. Установка финальных значений (par.c 690-696)

```
finalPrefix = prefix.HasValue ? prefix.Value : (numin > hang + 1 ? computedPrefix : augmentedFallbackPre)
finalSuffix = suffix.HasValue ? suffix.Value : (numin > hang + 1 ? computedSuffix : fallbackSuf)

return SetAffixesResult(finalPrefix, finalSuffix, augmentedFallbackPre, fallbackSuf)
```

> **Примечание:** `P` и `S` в `segments[startIndex].Prop` уже установлены
> вызовом `Delimit` (fallback prelen/suflen). Функция `setaffixes` их только читает.

### 8. Тесты (в xParTests, класс SetAffixesTests)

**Базовые тесты:**
1. Одна строка, prefix/suffix заданы → возвращает их
2. Одна строка, prefix/suffix = -1 → fallback из Prop
3. Две строки, prefix/suffix = -1 → compresuflen строк (без hang)

**Тесты с hang:**
4. Три строки, hang=1 → compresuflen строк [1..2]
5. Две строки, hang=1 → fallback (numin <= hang+1)

**Тесты quote:**
6. Одна строка, quote=true, есть quote-символы после fallbackPre → augmented
7. Одна строка, quote=false → augmented = fallbackPre

**Тесты с body:**
8. Две строки, body=true → compresuflen с учётом body-символов

### 9. Интеграция

Метод `SetAffixes` добавляется в `LineReader` как `public static`.
Вызывается после `Delimit` и `MarkSuperf`, перед основным циклом переформатирования.
