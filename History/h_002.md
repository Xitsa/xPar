# Шаг 2: Реализация обработки наборов символов (charset)

## Цель шага

Реализовать класс `Charset` для работы с наборами символов, соответствующий оригинальному `par` (Par 1.53.0), и добавить поддержку опций `B`, `P`, `Q`, `W`, `Z` в парсере аргументов.

## План шага

### 1. Создать класс `Charset` в проекте xParLib

Класс должен поддерживать:

**Charset syntax** (из par.doc):
- Обычные символы — добавляются в набор напрямую
- Escape-последовательности (начинаются с `_`):
  - `__` → underscore `_`
  - `_s` → space ` `
  - `_S` → все пробельные символы (класс `CharsetFlag.Space`)
  - `_b` → backslash `\`
  - `_q` → single quote `'`
  - `_Q` → double quote `"`
  - `_A` → все uppercase буквы (класс `CharsetFlag.UpperCase`)
  - `_a` → все lowercase буквы (класс `CharsetFlag.LowerCase`)
  - `_@` → все neither-case буквы (класс `CharsetFlag.NoCase`)
  - `_0` → все десятичные цифры Unicode (класс `CharsetFlag.Digit`)
  - `_xhh` → code point по hex-коду, конвертируется в строку (2 hex цифры, U+0000–U+00FF)
  - `_Xhhhh` → расширенный code point по hex-коду (4 hex цифры, U+0000–U+FFFF, включая supplementary planes через surrogate pairs)

**Методы класса:**
- `static Charset Parse(string str)` — разбирает строку в формате charset syntax
- `bool IsMember(string grapheme)` — проверяет принадлежность графем-кластера набору
- `Charset Union(Charset other)` — возвращает объединение двух множеств
- `Charset Difference(Charset other)` — возвращает разность множеств (this - other)
- `void Add(Charset other)` — добавляет графемы из другого множества (изменяет текущее)
- `void Remove(Charset other)` — удаляет графемы из текущего множества
- `Charset Clone()` — создаёт копию множества

**Внутренняя структура:**
- `_inList` — `HashSet<string>` графем-кластеров, которые точно в множестве
- `_outList` — `HashSet<string>` графем-кластеров, которые точно НЕ в множестве (для исключения)
- `_flags` — `HashSet<CharsetFlag>` флагов классов символов (UpperCase, LowerCase, NoCase, Digit, Space)

**Примечание о Unicode:**
- `IsMember(string grapheme)` проверяет: сначала `_inList`, потом `_outList`, потом `_flags`
- Для проверки флагов используется первый code point графемы через `CharUnicodeInfo.GetUnicodeCategory()`
- Графемы из нескольких code points проверяются по первому code point для флагов
- `_xhh` создаёт строку из одного code point (`char.ConvertFromUtf32(int)`)
- `_Xhhhh` создаёт строку из расширенного code point (может использовать surrogate pair)

### 2. Добавить свойства charset в `ParOptions`

Добавить следующие свойства:
- `BodyChars` (Charset) — набор символов тела
- `ProtectChars` (Charset) — набор защитных символов
- `QuoteChars` (Charset) — набор символов цитирования
- `WhiteChars` (Charset) — набор пробельных символов
- `TerminalChars` (Charset) — набор терминальных символов

**Инициализация charset из переменных окружения:**

В оригинальном `par` значения по умолчанию задаются в `main()` ДО вызова `parseargs()`:
```c
// Из par.c:
bodychars = parsecharset(getenv("PARBODY") ?? "");
protectchars = parsecharset(getenv("PARPROTECT") ?? "");
quotechars = parsecharset(getenv("PARQUOTE") ?? "> ");
whitechars = parsecharset(" \f\n\r\t\v");
terminalchars = parsecharset(".?!:");
```

**В xpar:**
- `ParOptions.Parse` принимает **уже инициализированные** `Charset` объекты
- Это позволит `Main()` сначала прочитать переменные окружения/конфигурацию, создать `Charset`,
  и передать их в `Parse`
- `Parse` будет модифицировать переданные charset'ы при обработке опций `B`, `P`, `Q`, `W`, `Z`
- **Все значения по умолчанию определяются в `Main()`, НЕ в `Parse`**

**Сигнатура Parse:**
```csharp
public static ParOptions Parse(
    string[] args,
    Charset bodyChars,
    Charset protectChars,
    Charset quoteChars,
    Charset whiteChars,
    Charset terminalChars)
```

Все charset — обязательные параметры (не null), создаются вызывающим кодом (Main).

### 3. Обновить метод `ParOptions.Parse` для обработки B, P, Q, W, Z

**Формат аргументов:**
- `B<op><set>` — body characters
- `P<op><set>` — protective characters
- `Q<op><set>` — quote characters
- `W<op><set>` — white characters
- `Z<op><set>` — terminal characters

Где `<op>` — один из:
- `=` — установить набор (заменить существующий)
- `+` — добавить символы к существующему набору
- `-` — удалить символы из существующего набора

`<set>` — строка в формате charset syntax

**Правила парсинга:**
- Опции `B`, `P`, `Q`, `W`, `Z` должны занимать целый аргумент командной строки (не могут комбинироваться с другими опциями)
- Если опция указана несколько раз с разными операторами, они применяются последовательно
- При `=` набор заменяется полностью, при `+` и `-` — модифицируется

### 4. Добавить тесты в xParTests

**Тесты для Charset.Parse:**
- Простые символы: `abc` → набор содержит "a", "b", "c"
- Escape-символы: `_s`, `_b`, `_q`, `_Q`, `__`
- Hex-символы: `_x41` ("A"), `_x7a` ("z")
- Расширенные hex: `_X00A9` ("©"), `_X2603` ("☃")
- Классы символов: `_A` (uppercase), `_a` (lowercase), `_0` (digits), `_S` (space)
- Комбинации: `_A_a_0` (alphanumeric), `._?!:` (terminal)
- Некорректный синтаксис: `_` в конце строки, `_x` без цифр, `_X` без цифр

**Тесты для IsMember:**
- `IsMember("a")` для набора с `_a` → true
- `IsMember("A")` для набора с `_A` → true
- `IsMember("©")` для набора с `_X00A9` → true
- `IsMember("☃")` для набора `_X2603` → true
- `IsMember(" ")` для набора `_S` → true
- `IsMember("ab")` для набора с "ab" → true (многосимвольная графема)
- **Кириллица:** `IsMember("ф")` для набора `_a` → true, `IsMember("Ф")` для набора `_A` → true
- **Unicode категории:** проверка что `_A`/`_a` работают не только для латиницы, но и для кириллицы, греческого и других алфавитов

**Тесты для операций над множествами:**
- `IsMember` для символов из inList и flags
- `Union` — объединение двух множеств
- `Difference` — разность двух множеств
- `Add` и `Remove` — мутация множества

**Тесты для ParOptions.Parse с charset:**
- `B=abc` — устанавливает bodychars
- `P+_A` — добавляет uppercase к protectchars
- `Q-_s` — удаляет space из quotechars
- `W=_S` — устанавливает все whitespace
- `Z=.?!:` — устанавливает terminalchars
- Несколько операций: `B=abc`, `B+_d` → bodychars = "a", "b", "c", "d"

## Результат шага

- Создан класс `Charset` в xParLib с полной поддержкой charset syntax:
  - Escape-последовательности: `__`, `_s`, `_b`, `_q`, `_Q`, `_xhh`, `_Xhhhh`
  - Классы символов: `_A` (uppercase), `_a` (lowercase), `_0` (digits), `_S` (space), `_@` (nocase)
  - Работает с grapheme clusters (строками C#), а не с отдельными char
  - Корректно обрабатывает Unicode (кириллица, греческий и др.)

- Реализованы операции над множествами:
  - `Union` — объединение двух множеств

  - `Remove` — удаление символов
  - `Clone` — создание независимой копии
  - `Replace` — полная замена содержимого
