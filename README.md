# xpar — Консольное приложение для переформатирования абзацев

## Описание проекта

**xpar** — это консольное приложение на C#, являющееся функциональной копией утилиты `par` (версия 1.53.0).
Оно читает строки из стандартного ввода, переформатирует абзацы согласно заданным параметрам и выводит результат.
Проект реализует поддержку Unicode через работу с графем-кластерами, также поддерживается расчёт длин строк с учётом визуальной ширины символов.

## Причины появления порта утилиты

`par` долго  шёл рядом  со мной по  жизни, ещё  начиная с `MS  DOS` и  далее в
`Windows`.  Но постепенно  в  мире распространялся  `Unicode` и  мультибайтные
кодировки.  С  кодировками можно  было  бороться,  переключая кодировку  файла
на  время в  однобайтную,  но с  распространением  символов, которые  занимали
несколько кодовых позиций в строке, `par` всё более переставал справляться.

Патчи  Jérôme  Pouiller’а мне  не  удалось  запустить под  `Windows`,  поэтому
постепенно  перестал эту  утилиту использовать,  хоть и  всё время  её мне  не
хватало. Я  несколько порывался  портировать её на  другие языки  с поддержкой
`Unicode`, но сил не хватало 😞.

В этот раз я решил попробовать заново, но со следующими ограничениями:

*   поддержка `Unicode` изначально, работа с кластерами графем;
*   упор на повторение оригинальной логики форматирования;
*   отказ  от производительности:  `par` довольно  экономно расходует  память,
    я  пожертвовал   этим  ради  удобства  отделения   методов  и  возможности
    независимого тестирования;
*   поддержка только `utf-8` кодировки, так как я других сейчас не использую и
    мне так удобнее.

## Основные компоненты

### xPar/Program.cs
- Точка входа `Main`.
- Считывает **все** строки из `stdin` до EOF/завершения ввода.
- Формирует `IReadOnlyList<string>` и передаёт его в `StringTransformer.Transform` вместе с параметрами (`ParOptions`).
- Выводит отформатированные строки в `stdout`.

### xParLib/StringTransformer.cs
- Публичный класс `StringTransformer`.
- Публичный метод `Transform(IReadOnlyList<string>, ParOptions)` — полный конвейер:
  1. `LineReader.ReadLines` — разбор входных строк и их аннотирование.
  2. `LineReader.Delimit` — определение бестелесных строк и разметка свойств.
  3. `LineReader.MarkSuperf` — разметка избыточных строк.
  4. `LineReader.SetAffixes` — вычисление финальных префикса и суффикса.
  5. `ReformatModule.Reformat` — непосредственное форматирование абзацев.
  6. `ConstructLines` — построение выходных строк с учётом префиксов/суффиксов.
- Все операции выполняются с учётом графем (grapheme clusters) и их визуальной ширины.

### xParLib/Reformat.cs
- Содержит класс `ReformatModule` с методами:
  - `ExtractWords` — выделение слов, обработка `guess`, проверка длины.
  - `SimpleBreaks`, `NormalBreaks`, `JustBreaks` — выбор разрывов строк для разных режимов выравнивания.
  - `ConstructLines` — формирование строк с префиксами/суффиксами.
- Использует `StringInfo.GetTextElementEnumerator()` и `Wcwidth.UnicodeCalculator.GetWidth()` для корректной работы с Unicode.

### xParLib/ParOptions.cs
- Класс, хранящий параметры форматирования (width, tab, charset и др.).
- Статический метод `Parse(string[] args)` — разбор аргументов командной строки.
- Параметры по умолчанию совпадают с оригинальным `par`.

### xParLib/Charset.cs
- Обеспечивает работу с набором символов (`charset`–syntax).
- Методы `Parse(string)`, `IsMember(string)`, `Union`, `Difference`.
- Поддерживает escape–последовательности (`_A`, `_a`, `_0`, `_S`, `_xHH`, `_Xhhhh`).

### xParLib/LineReader.cs
- Содержит методы:
  - `ReadLines` — чтение строк до EOF/blank/protected line.
  - `Compresuflen` — вычисление общих префикса и суффикса.
  - `Delimit` — определение бестелесных строк, разметка свойств.
  - `MarkSuperf` — разметка избыточных строк.
  - `SetAffixes` — вычисление финальных префикса и суффикса.
- Вспомогательные методы: `IsBodiless`, `IsInserted`, `IsVacant`, `CountGraphemes`, `CountNonSpaceGraphemes`.

## Как использовать

1. **Сборка проекта**
   ```bash
   dotnet build
   ```

2. **Запуск приложения**
   ```bash
   dotnet run --project xPar
   ```
   Вводятся строки по одной, каждая с новой строки. Для завершения ввода используйте `Ctrl+Z` (Windows) или `Ctrl+D` (Linux/macOS).

3. **Пример вывода**
   ```
   Hello World[5, 5]
   Test[4]
   ```

4. **Запуск тестов**
   ```bash
   dotnet test --project xParTests
   ```

## Особенности реализации

- Вводятся **все** строки перед обработкой; позиция чтения не сохраняется между вызовами (в отличие от оригинального `par`).
- Для корректной работы с Unicode используется `StringInfo.GetTextElementEnumerator()` и `Wcwidth.UnicodeCalculator.GetWidth()`.
- Текст нормализуется в форму **KD** перед дальнейшей обработкой.
- Публичные методы и классы снабжены полным XML–документам на русском языке.
- История изменений описана в каталоге `History/` (файлы `h_001.md`, `h_002.md` и т.д.).
